# Тестирование Papa Pizza

Дипломный проект по автоматизации тестирования страницы сайта "Papa Pizza" ("http://papapizza59.ru/).  
Проект реализован на языке **Python** с использованием фреймворка **pytest** и библиотеки **Selenium WebDriver**.


# Начало работы 

Перед началом работы установить **requirements.txt**. Ввести команду в терминал: `pip install -r requirements.txt`.

# Запуск автоматизированных тестов

Для запуска автоматизированных тестов использовать команду в терминале: `pytest -m papa_pizza`.
Для запуска автоматизированных тестов allure использовать команду в терминале:`pytest -m papa_pizza --alluredir=allure-results`.
Для открытия отчета allure использовать команду в терминале: `allure serve allure_result`.

## Пример автоматизированного теста (test_papa_pizza_image.py)

```
# ─────────────────────────────────────────────────────────────────────
# 1. Импорт библиотек
# ─────────────────────────────────────────────────────────────────────
import allure                          # 🔹 Библиотека для красивых отчётов
import pytest                          # 🔹 Фреймворк для запуска тестов
from pages.page_main import LoginPage  # 🔹 Импорт класса страницы (Page Object)

# ─────────────────────────────────────────────────────────────────────
# 2. Allure-аннотации (метаданные для отчёта)
# ─────────────────────────────────────────────────────────────────────
@allure.id("002")                      # 🔹 Уникальный номер теста (как в тест-кейсе)
@allure.label("Papa pizza")            # 🔹 Метка проекта/компонента
@allure.title("Papa pizza проверка изображения")  # 🔹 Название теста в отчёте
@allure.description("Тест проверяет изображения на главной странице, что оно отсылает на начальную страницу")
                                       # 🔹 Подробное описание что проверяем

# ─────────────────────────────────────────────────────────────────────
# 3. Pytest-маркеры (для фильтрации и параметризации)
# ─────────────────────────────────────────────────────────────────────
@pytest.mark.papa_pizza                # 🔹 Маркер для запуска только тестов Papa Pizza
@pytest.mark.parametrize("driver", ["edge", "chrome"], indirect=True)
                                       # 🔹 Запуск теста в 2 браузерах: Edge и Chrome
                                       # 🔹 indirect=True означает, что driver берётся из conftest.py

# ─────────────────────────────────────────────────────────────────────
# 4. Объявление функции теста
# ─────────────────────────────────────────────────────────────────────
def tests_002(driver):                 # 🔹 Название теста (должно начинаться с test_)
                                       # 🔹 driver — фикстура из conftest.py (браузер)

# ─────────────────────────────────────────────────────────────────────
# 5. Инициализация Page Object
# ─────────────────────────────────────────────────────────────────────
    login = LoginPage(driver)          # 🔹 Создаём объект страницы LoginPage
                                       # 🔹 Передаём ему driver для управления браузером

# ─────────────────────────────────────────────────────────────────────
# 6. Шаг 1: Открытие сайта
# ─────────────────────────────────────────────────────────────────────
    with allure.step("Открываем сайт Papa pizza"):  # 🔹 Начало шага в отчёте Allure
        login.open()                                # 🔹 Метод открывает сайт (driver.get())

# ─────────────────────────────────────────────────────────────────────
# 7. Шаг 2: Закрытие модального окна
# ─────────────────────────────────────────────────────────────────────
    with allure.step('Кликнуть "Да" в модальном окне'):  # 🔹 Название шага в отчёте
        login.click_modal_window()                       # 🔹 Клик по кнопке в модальном окне

# ─────────────────────────────────────────────────────────────────────
# 8. Шаг 3: Переход в раздел "Пицца"
# ─────────────────────────────────────────────────────────────────────
    with allure.step('Кликнуть в навигационном меню на кнопку "Пицца"'):
        login.click_pizza_button_navbar()  # 🔹 Клик по пункту меню "Пицца"

# ─────────────────────────────────────────────────────────────────────
# 9. Шаг 4: Проверка перехода в раздел "Пицца"
# ─────────────────────────────────────────────────────────────────────
    with allure.step('Проверяем что мы во вкладке "Пицца"'):
        assert login.wait_pizza_header(), "ОШИБКА не перешел по вкладке пицца"
        # 🔹 assert — проверка условия
        # 🔹 wait_pizza_header() — ждёт заголовок раздела "Пицца"
        # 🔹 Если False → тест упадёт с сообщением об ошибке

# ─────────────────────────────────────────────────────────────────────
# 10. Шаг 5: Клик по логотипу
# ─────────────────────────────────────────────────────────────────────
    with allure.step('Кликнуть на изображение сверху "PAPA PIZZA"'):
        login.click_main_image()  # 🔹 Клик по логотипу (кнопка "Домой")

# ─────────────────────────────────────────────────────────────────────
# 11. Шаг 6: Проверка возврата на главную
# ─────────────────────────────────────────────────────────────────────
    with allure.step('Проверяем что мы на начальной странице'):
        assert login.wait_main_title(), "ОШИБКА не перешел на начальную страницу"
        # 🔹 Проверяем, что после клика по логотипу вернулись на главную
        # 🔹 Если не вернулись → тест FAILED с этим сообщением
```
